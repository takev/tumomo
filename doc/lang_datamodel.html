Taula data model

by Take Vos (2013)

Objects are Taula's abstraction for all data. All data in a Taula program is represented by objects or by relations between objects.

There are three different kind of objects:

  • Primative types.
  • Arrays of objects.
  • Classes.

Memory management

Primatives and vectors (and classes which only contain a single primative or vector) can be held in registers and may not be stored in memory at all. If there are not enough registers the compiler will allocate extra memory on the stack to store these.

If stack size permits, objects that are used locally will be allocated on the stack.

Objects which are stored in the global namespace may be allocated statically.

A dynamic sized object returned by a function must be allocated on the heap.
